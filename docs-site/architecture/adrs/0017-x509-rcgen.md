# ADR 0017 — X.509 certificate generation: rcgen

Constrains: FR-FLEET-009, ADR-0009

Date: 2026-04-19
Status: Accepted

## Context

`hwledger-server mint-admin-cert` and agent CSR signing (ADR-0009) require generating X.509 certs in-process: the admin MVP self-signs, and agents eventually CSR against a CA owned by the server. The library must produce rustls-compatible PEM/DER, support ed25519/ECDSA P-256, handle SANs (DNS + IP), and not shell out to OpenSSL.

## Options

| Option | In-process | Ed25519 | ECDSA P-256 | SANs | Rustls-compatible | Surface area |
|---|---|---|---|---|---|---|
| rcgen 0.13 | Yes | Yes | Yes | Yes | Yes | Small, clean API |
| OpenSSL subprocess | No | Yes | Yes | Yes (awkward) | Yes | Huge, shell-deps |
| picky-rs | Yes | Yes | Yes | Yes | Partial (PKCS#7 focus) | Medium |
| `x509-certificate` | Yes | Yes | Yes | Manual | Yes | Low-level, lots of boilerplate |
| bcder hand-rolled | Yes | Yes | Yes | Manual | Yes | Very low-level |

## Decision

Use **rcgen 0.13** for all cert generation (admin cert, agent CSR, self-signed CA). rustls consumes the DER output directly via `rustls-pemfile`.

## Rationale

- Highest-level API of the pure-Rust options. `Certificate::generate_self_signed(vec!["admin"])` is a one-liner.
- Actively maintained by the rustls team (shared maintainers); version alignment with rustls is explicit.
- Supports every extension we need (SANs, KeyUsage, ExtendedKeyUsage, BasicConstraints::is_ca).
- PKCS#12 bundling is missing but trivially added via `p12-keystore` for the MVP.

## Consequences

- We inherit rcgen's defaults (SHA-256, 2048-bit RSA or ed25519). Any policy drift (e.g., requiring SHA-384) requires a version pin and release note.
- PKCS#12 export is a second dep; acceptable.

## Revisit when

- rcgen adds native PKCS#12 or gets deprecated.
- FIPS-140 compliance becomes a requirement (pure-Rust cert libs do not claim FIPS).
