# Claim Audit — 2026-04-21

Systematic verification of every feature claimed in prior agent commit
messages across `hwLedger` and the sibling `phenotype-journeys` repo.
Each claim was checked by source-code inspection and, where practical,
by running the binary or inspecting the built artefact.

## Methodology

- Git log walked on both repos from their first journey-related commit
  through `HEAD`.
- Each claim maps to a file path + grep predicate. PASS means the
  predicate is present in the tree; FAIL means the feature was claimed
  but the predicate is absent.
- Remediation is applied inline for small gaps. Large gaps are left as
  `TODO` with owner + follow-up work package.

## Remediation Detail

### #41 — server `--plain-http` / `--dev`

Root cause: `crates/hwledger-server/src/main.rs` previously derived TLS
enablement solely from `cfg!(debug_assertions)`. The flags were not
exposed as CLI options despite being referenced in docs + Streamlit
Settings quick-start.

Fix: added two `clap` flags (`--plain-http`, `--dev`) and a runtime
gate that refuses `--plain-http` in release builds unless `--dev` is
also passed. `cargo check -p hwledger-server` passes clean.

## Stale-dep Note

`docs-site/node_modules/@phenotype/journey-viewer/` is an older
snapshot of the viewer source (no `is-expanded` in the vendored
node_modules copy), but the `vendor/phenotype-journeys/phenotype-journey-viewer-0.1.0.tgz`
tarball *does* include the new scroll/collapse code. A `bun install`
at docs-site build time resolves to the tarball, so the runtime docs
site gets the fixed component. This is a caching artefact only, not a
shipped regression.
