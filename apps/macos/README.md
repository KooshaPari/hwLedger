# hwLedger macOS

Swift/SwiftUI integration for hwLedger via XCFramework + Swift Package.

## Prerequisites

- macOS 14+ (Sonoma or later)
- Xcode 16+
- Rust 1.70+ with Apple targets installed:
  ```bash
  rustup target add aarch64-apple-darwin x86_64-apple-darwin
  ```
- cbindgen installed:
  ```bash
  cargo install cbindgen
  ```

## Building the XCFramework

From the project root:

```bash
./scripts/build-xcframework.sh --release
```

This produces:
- `apps/macos/xcframework/HwLedgerCore.xcframework` — Universal binary (arm64 + x86_64)
- `apps/macos/xcframework/build/include/hwledger.h` — C header (generated by cbindgen)

### Build modes

```bash
./scripts/build-xcframework.sh --release  # Optimized (default)
./scripts/build-xcframework.sh --debug    # With debug symbols
```

### What the script does

1. Compiles `hwledger-ffi` for both `aarch64-apple-darwin` and `x86_64-apple-darwin`
2. Creates a universal static library using `lipo`
3. Generates C header using `cbindgen`
4. Writes a `module.modulemap` for Clang module integration
5. Builds an XCFramework bundle via `xcodebuild -create-xcframework`

## Using in Xcode

### Option 1: Drag-and-Drop (Swift Package + XCFramework)

1. Build the XCFramework:
   ```bash
   ./scripts/build-xcframework.sh --release
   ```

2. In your Xcode project, open **File > Add Packages**

3. Add the local path to `hwledger-swift`:
   ```
   /path/to/hwledger/apps/macos/hwledger-swift
   ```

4. In **Build Phases**, add `HwLedgerCore.xcframework` to **Link Binary with Libraries**

5. Import and use:
   ```swift
   import HwLedger

   let version = HwLedger.coreVersion()
   let devices = try HwLedger.detectDevices()
   let plan = try HwLedger.plan(
       configJson: modelConfig,
       seqLen: 4096,
       concurrentUsers: 2,
       batchSize: 1
   )
   ```

### Option 2: Swift Package Manager (Remote)

Once published to a public registry, you can reference it directly:

```swift
dependencies: [
    .package(url: "https://github.com/KooshaPari/hwledger.git", from: "0.1.0")
],
targets: [
    .target(dependencies: [.product(name: "HwLedger", package: "hwledger")])
]
```

## API Overview

### Core Functions

- `HwLedger.coreVersion() -> String` — Get FFI version string
- `HwLedger.plan(configJson:seqLen:concurrentUsers:batchSize:kvQuantization:weightQuantization) throws -> PlannerResult` — Plan memory for a model
- `HwLedger.detectDevices() throws -> [DeviceInfo]` — Enumerate all GPUs
- `HwLedger.sample(deviceId:backend:) throws -> TelemetrySample` — Sample GPU telemetry

### Types

#### PlannerResult
```swift
struct PlannerResult {
    var weightsBytes: UInt64
    var kvBytes: UInt64
    var prefillActivationBytes: UInt64
    var runtimeOverheadBytes: UInt64
    var totalBytes: UInt64
    var attentionKindLabel: String  // "Mha", "Gqa", "Mla", etc.
    var effectiveBatch: UInt32
}
```

#### DeviceInfo
```swift
struct DeviceInfo {
    var id: UInt32
    var backend: String              // "nvidia", "amd", "metal", etc.
    var name: String                 // Device name
    var uuid: String                 // Device UUID (if available)
    var totalVramBytes: UInt64
}
```

#### TelemetrySample
```swift
struct TelemetrySample {
    var deviceId: UInt32
    var freeVramBytes: UInt64
    var utilizationPercent: Float    // 0.0-100.0
    var temperatureCelsius: Float
    var powerWatts: Float
    var capturedAtMs: UInt64         // UNIX epoch milliseconds
}
```

#### Quantization Enums
```swift
enum KvQuantization: Int32 {
    case fp16 = 0
    case fp8 = 1
    case int8 = 2
    case int4 = 3
    case threeBit = 4
}

enum WeightQuantization: Int32 {
    case fp16 = 0
    case bf16 = 1
    case int8 = 2
    case int4 = 3
    case threeBit = 4
}
```

## Example Usage

```swift
import HwLedger

// Check version
print("hwLedger version: \(HwLedger.coreVersion())")

// Detect GPUs
do {
    let devices = try HwLedger.detectDevices()
    for device in devices {
        print("Device \(device.id): \(device.name) (\(device.backend))")
    }
} catch {
    print("Failed to detect devices: \(error)")
}

// Plan memory for DeepSeek-V3
let config = """
{
  "model_type": "deepseek",
  "num_hidden_layers": 62,
  "hidden_size": 4096,
  "kv_lora_rank": 512,
  "qk_rope_head_dim": 64
}
"""

do {
    let result = try HwLedger.plan(
        configJson: config,
        seqLen: 4096,
        concurrentUsers: 2,
        batchSize: 1,
        kvQuantization: .fp16,
        weightQuantization: .int4
    )
    print("Total memory needed: \(result.totalBytes / (1024*1024*1024))GB")
    print("Attention type: \(result.attentionKindLabel)")
} catch {
    print("Planning failed: \(error)")
}

// Sample GPU telemetry
if let device = devices.first {
    do {
        let sample = try HwLedger.sample(deviceId: device.id, backend: device.backend)
        print("GPU utilization: \(sample.utilizationPercent)%")
        print("Temperature: \(sample.temperatureCelsius)C")
        print("Free VRAM: \(sample.freeVramBytes / (1024*1024))MB")
    } catch {
        print("Sampling failed: \(error)")
    }
}
```

## Troubleshooting

### "command not found: cargo"
Install Rust: `curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh`

### "error: can't find crate for 'HwLedgerCore'"
The XCFramework was not built. Run: `./scripts/build-xcframework.sh --release`

### Linker errors on arm64 only / x86_64 only
Verify the universal binary: `lipo -info apps/macos/xcframework/build/universal/libhwledger_ffi.a`
Should list: `Architectures in the fat file: arm64 x86_64`

### "module.modulemap not found"
Regenerate with the build script. Ensure `xcodebuild` is available.

## Architecture Notes

The build uses:
- **cbindgen** to generate C header from Rust `#[repr(C)]` types
- **lipo** to create a universal binary (arm64 + x86_64)
- **xcodebuild** to bundle into XCFramework format
- **Swift FFI** (via `@_silgen_name`) to call raw C functions

All memory is freed explicitly in Swift wrapper destructors to prevent leaks.
